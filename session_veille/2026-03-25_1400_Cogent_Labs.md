## 2. Probleme adresse

Le marche japonais (et asiatique) repose encore massivement sur des documents papier, manuscrits et non-structures (factures, contrats, formulaires, questionnaires, livrets bancaires). La saisie manuelle de ces documents est chronophage, couteuse et source d'erreurs. Cogent Labs adresse le probleme de la digitalisation et de l'extraction automatique de donnees a partir de documents complexes, notamment en japonais (caracteres kanji, hiragana, katakana) qui representent un defi technique majeur pour l'OCR classique.

## 5. Solutions proposees

### SmartRead (produit phare)
- **Type** : Plateforme IDP (Intelligent Document Processing) / AI-OCR de nouvelle generation
- **Precision** : 99,2% de reconnaissance, y compris caracteres manuscrits
- **Technologies** : OCR propriétaire, IA generative (upgrade mai 2024), NLP, Deep Learning
- **7 ans de R&D** sur une IA proprietaire specialisee pour les documents japonais
- **Fonctionnalites cles** :
  - Lecture automatique multi-formats : factures, bons de commande, contrats, questionnaires, livrets bancaires
  - Extraction sans template (depuis l'upgrade IA generative 2024) - traitement de documents non-structures
  - SmartVerify : verification et correction automatiques (-90% temps de conversion)
  - Mode SaaS et On-Premises (pour environnements securises)
  - Connecteurs RPA : UiPath Marketplace, WinActor
  - API d'integration avec systemes metier tiers

### SmartFind (nouveau produit 2025)
- Interface conversationnelle type ChatGPT pour la recherche documentaire
- Recherche intelligente dans les corpus de documents

### Kaidoku
- Service de traitement du langage naturel (NLP)
- Analyse et comprehension de textes pour ameliorer l'efficacite operationnelle

### Cas d'usage assurance
- SmartRead supporte les documents d'assurance (mentione explicitement dans les secteurs cibles post-upgrade IA generative 2024)
- Extraction automatique de declarations de sinistres, formulaires de souscription, contrats
- Applicable a la digitalisation de dossiers sinistres papier, formulaires manuscrits de declaration
- Integration possible dans les chaines de traitement automatise des sinistres via connecteurs RPA
