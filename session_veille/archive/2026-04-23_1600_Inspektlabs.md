## 1. Presentation generale

- **Nom** : Inspektlabs (Inspekt Labs Inc.)
- **Siege social** : New Castle, Delaware, USA
- **Centres de developpement** : Delhi et Jaipur (Inde)
- **Annee de creation** : 2019
- **Secteur** : Insurtech / ClaimsTech - Inspection automatisee de vehicules par IA
- **Modele** : B2B SaaS / Inspection-as-a-Service
- **Site web** : https://inspektlabs.com/
- **Mission** : Automatiser les inspections d'actifs physiques (principalement vehicules) a partir de photos et de videos capturees par smartphone, pour les assureurs auto, gestionnaires de flotte, loueurs et reseaux de reparation.

## 2. Probleme adresse

L'inspection manuelle de vehicules (pour souscription, renouvellement, sinistre, remarketing, retour de location) est chronophage, couteuse, subjective et vulnerable a la fraude. Elle necessite souvent le deplacement d'experts ou l'immobilisation du vehicule. Inspektlabs propose de remplacer ce processus par une capture photo/video par smartphone + analyse IA en quelques secondes, generant un rapport de dommages chiffre, avec detection de fraude integree.

## 5. Solutions proposees

### Plateforme Inspektlabs (produit phare)
- **Type** : Plateforme SaaS d'inspection vehicule par IA (photo + video)
- **Precision** : 95-99% de precision sur la detection de dommages
- **Couverture** : 21 types de dommages sur 163 pieces du vehicule
- **Entrainement IA** : 30M+ images de dommages reels (certains chiffres indiquent > 1M images/videos dans les communications Guidewire Vanguards)
- **Vehicules supportes** : voitures, camions, motos

### Modules principaux
- **Photo/Video Capture** : application mobile et SDK de capture guidee (360 video, conseils en temps reel sur la qualite) ; pas de materiel additionnel requis, fonctionne avec smartphone standard
- **Damage Detection** : detection automatique de rayures, bosses, fissures, pieces cassees
- **Claim Estimation** : calcul automatique d'un devis de reparation aligne sur les pratiques locales de reparation
- **Fraud Detection** : detection en temps reel d'images digitalement modifiees, de photos dupliquees, de submissions de mauvaise qualite, de tentatives de dissimuler des dommages anterieurs
- **Underwriting / PIV (Pre-Inspection Verification)** : inspection pour souscription ou renouvellement de police
- **Workflow / API** : API REST + webhooks (payload JSON avec URLs vehicules et coordonnees des dommages) + web app + native app + SDK integrables aux systemes existants

### Cas d'usage assurance et au-dela
- **Motor Insurance** : claims, souscription, renouvellement, detection de fraude
- **Fleet Management** : suivi de condition, inspections recurrentes des flottes
- **Car Rental** : inspections pre/post-trip, etats des lieux digitaux
- **Leasing** : inspections de retour de lease
- **Remarketing automobile** : rapports de condition pour la revente
- **Reseaux de reparation** : estimation automatisee
- **Extensions futures** : telephones mobiles, velos, infrastructures (tuyaux, fuuts de petrole...)

### Workflow type
1. L'assure/agent capture photos ou video 360 avec l'application (guidance en temps reel)
2. Upload cloud
3. Traitement IA : detection des dommages, identification des pieces, chiffrage, scoring fraude
4. Rapport complet retourne en quelques secondes (JSON + PDF)
5. Si dommages dans les seuils predefinis, approbation automatique du sinistre sans intervention humaine
6. Integration directe dans le SI de l'assureur via webhook

### Delai d'integration
- La plupart des clients completent l'integration en 1 a 2 semaines (API setup, formation, personnalisation workflow)
